Вопросы для самопроверки

1)Что такое managed и unmanaged ресурсы?
2)Приведите примеры unmanaged ресурсов.
3)Что такое IDisposable и как он применяется?
4)Какими дополнительными свойствами обладает класс, реализовывающий интерфейс IDisposable? (вопрос про using)
5)Что такое using и в какой код он разворачивается?
6)Приведите пример использования деструктора (финализатора).
7)Что делает вызов GC.SurpressFinalize()?

1) Глобально переменные можно разделить на managed и unmanaged. Managed ресурсы - ресурсы, что очистятся сборщиком мусора по ходу выполнения программы, если на них никто не ссылается или по ее завершению. Unmanaged ресурсы в свою очередь не очищаются сборщиком мусора и разработчик сам решает, как и когда они должны это делать.
2) В unmanaged ресурсы входят - ресурсы для работы с файлами, подключением к базам данных, сетевое взаимодействие и т.д.  
3) IDisposable интерфейс, который позволяет реализовать метод Dispose. В свою очередь данный метод нужен для unmanaged ресурсов, чтобы освободить, чтобы освободить место в памяти. 
4) Есть блок using, который можно вызвать только типами реализующие IDisposable.
5) Более удобная версия записи try/finally, которая гарантирует выполнения метода Dispose.
6) Деструктор(Финализатор), записывается как название класса с ~, нужен для сборщика мусора. Можно применить например для закрытия чтения файла.
7) GC.SurpressFinalize() означает, что сборщику мусора не нужно очищать данный файл. Зачастую используется в методе Dispose
